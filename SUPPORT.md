# Support

## Before Opening An Issue

- read [README.md](./README.md)
- check [CONTRIBUTING.md](./CONTRIBUTING.md)
- confirm your local configuration is coming from your own `.env`
- include the automation name, runner command, and exact failure mode

## Good Bug Reports

Include:

- the command you ran
- whether the run was live or fixture-based
- the relevant artifact JSON if one was produced
- whether SQLite or Postgres was configured
- whether the failure was a page-shape drift, auth drift, or persistence error

## Security Issues

For sensitive problems such as leaked credentials or bypasses, follow [SECURITY.md](./SECURITY.md) instead of opening a public issue.
